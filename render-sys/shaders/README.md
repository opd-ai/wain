# UI Shaders for GPU Rendering

This directory contains WGSL (WebGPU Shading Language) shaders for common UI draw operations. These shaders were authored as part of Phase 4.2 of the wain project and will be lowered to Intel EU machine code in Phase 4.3.

## Overview

All shaders follow a common pattern:
- **Vertex shader** (`vs_main`): Generates a full-screen quad from vertex index (0-3) using bit manipulation
- **Fragment shader** (`fs_main`): Performs the actual rendering operation

Each shader is designed to be simple, GPU-friendly, and suitable for UI rendering workloads.

## 1. solid_fill.wgsl

**Purpose**: Render a quad with a constant color (used for solid fills, backgrounds, borders).

### Entry Points
- `vs_main`: Vertex shader
- `fs_main`: Fragment shader

### Uniforms/Bindings
*Current implementation*: Hardcoded white color `vec4(1.0, 1.0, 1.0, 1.0)`

*Planned (Phase 4.3+)*: Push constant for color
```wgsl
struct PushConstants {
    color: vec4<f32>,  // RGBA color
}
```

### Vertex Format
None required — quad is generated procedurally from `vertex_index` (0-3).

### Expected Usage
```
Draw 4 vertices as triangle strip
Vertex 0: (-1, -1) → bottom-left
Vertex 1: ( 1, -1) → bottom-right
Vertex 2: (-1,  1) → top-left
Vertex 3: ( 1,  1) → top-right
```

### Output
Writes solid color to framebuffer with alpha blending support.

## 3. sdf_text.wgsl

**Purpose**: Render crisp text at any scale using signed distance fields.

### Entry Points
- `vs_main`: Vertex shader
- `fs_main`: Fragment shader

### Uniforms/Bindings
*Current implementation*: Hardcoded white color

*Planned (Phase 4.3+)*:
```wgsl
@group(0) @binding(0)
var sdf_atlas: texture_2d<f32>;  // SDF font atlas (single-channel)

@group(0) @binding(1)
var sampler: sampler;  // Linear filtering

struct TextParams {
    color: vec4<f32>,        // Text color
    threshold: f32,          // SDF edge threshold (default: 0.5)
    smoothing: f32,          // Anti-aliasing smoothness (default: 0.05)
    padding: vec2<f32>,
}

@group(1) @binding(0)
var<uniform> params: TextParams;
```

### Vertex Format
For positioned glyphs:
```wgsl
struct Vertex {
    position: vec2<f32>,  // Screen-space position
    uv: vec2<f32>,        // Atlas UV coordinates
}
```

### Algorithm
```wgsl
let distance = textureSample(sdf_atlas, sampler, uv).r;
let alpha = smoothstep(threshold - smoothing, threshold + smoothing, distance);
return vec4(color.rgb, color.a * alpha);
```

### Expected Usage
```
Bind SDF atlas texture (generated by cmd/gen-atlas)
Set threshold and smoothing parameters
Draw quads for each glyph with correct atlas UVs
```

### Output
Anti-aliased text with configurable edge smoothness.

## 4. box_shadow.wgsl

**Purpose**: Render Gaussian-blurred box shadows using separable two-pass blur.

### Entry Points
- `vs_main`: Vertex shader
- `fs_main`: Fragment shader (runs twice: horizontal pass, then vertical pass)

### Uniforms/Bindings
*Current implementation*: Hardcoded semi-transparent black `vec4(0, 0, 0, 0.5)`

*Planned (Phase 4.3+)*:
```wgsl
@group(0) @binding(0)
var input_texture: texture_2d<f32>;  // Source to blur

@group(0) @binding(1)
var sampler: sampler;  // Linear filtering

struct BlurParams {
    direction: vec2<f32>,   // (1, 0) for horizontal, (0, 1) for vertical
    kernel_size: u32,       // Number of samples (must be odd)
    sigma: f32,             // Gaussian blur standard deviation
    padding: f32,
}

@group(1) @binding(0)
var<uniform> params: BlurParams;
```

### Vertex Format
None required — generates full-screen quad.

### Algorithm (Separable Gaussian)
```wgsl
// Horizontal pass
for i in 0..kernel_size {
    let offset = direction * (i - kernel_size / 2);
    let weight = gaussian(offset, sigma);
    color += textureSample(input, sampler, uv + offset) * weight;
}

// Vertical pass uses same logic with direction = (0, 1)
```

### Expected Usage
```
1. Render source to texture A
2. Horizontal pass: A → texture B (direction = (1, 0))
3. Vertical pass: B → output (direction = (0, 1))
```

### Output
Gaussian-blurred shadow suitable for box shadow effects.

## 5. rounded_rect.wgsl

**Purpose**: Render rounded rectangles with SDF-based corner clipping.

### Entry Points
- `vs_main`: Vertex shader
- `fs_main`: Fragment shader

### Uniforms/Bindings
*Current implementation*: Hardcoded white fill

*Planned (Phase 4.3+)*:
```wgsl
struct RoundedRectParams {
    rect_size: vec2<f32>,      // Rectangle dimensions in pixels
    corner_radius: vec4<f32>,  // (top-left, top-right, bottom-right, bottom-left)
    fill_color: vec4<f32>,     // Interior color
    border_color: vec4<f32>,   // Border color
    border_width: f32,         // Border thickness in pixels
    smoothness: f32,           // Anti-aliasing edge softness
    padding: vec2<f32>,
}

@group(0) @binding(0)
var<uniform> params: RoundedRectParams;
```

### Vertex Format
Standard quad with UV mapping to pixel coordinates.

### Algorithm (SDF-based)
```wgsl
// Compute signed distance from rounded rect boundary
fn rounded_rect_sdf(pos: vec2<f32>, size: vec2<f32>, radius: f32) -> f32 {
    let q = abs(pos) - size + radius;
    return min(max(q.x, q.y), 0.0) + length(max(q, vec2(0.0))) - radius;
}

let sdf = rounded_rect_sdf(uv - 0.5, rect_size, corner_radius);
let alpha = smoothstep(smoothness, -smoothness, sdf);  // Interior fill
// Border logic: smoothstep(sdf, sdf - border_width)
```

### Expected Usage
```
Set rectangle size and corner radii
Draw quad covering rectangle bounds
SDF discards pixels outside rounded corners
```

### Output
Anti-aliased rounded rectangle with optional border.

## 6. linear_gradient.wgsl

**Purpose**: Render linear color gradients along a direction vector.

### Entry Points
- `vs_main`: Vertex shader
- `fs_main`: Fragment shader

### Uniforms/Bindings
*Current implementation*: Simple horizontal grayscale gradient

*Planned (Phase 4.3+)*:
```wgsl
struct GradientStop {
    position: f32,      // Normalized position [0, 1]
    color: vec4<f32>,   // RGBA color
}

struct LinearGradientParams {
    start_point: vec2<f32>,  // Gradient start in UV space
    end_point: vec2<f32>,    // Gradient end in UV space
    stop_count: u32,         // Number of color stops (max 8)
    padding: vec3<f32>,
}

@group(0) @binding(0)
var<uniform> params: LinearGradientParams;

@group(0) @binding(1)
var<storage, read> stops: array<GradientStop>;  // Max 8 stops
```

### Vertex Format
Standard quad with UV coordinates.

### Algorithm
```wgsl
let direction = end_point - start_point;
let t = dot(uv - start_point, direction) / dot(direction, direction);
let clamped_t = clamp(t, 0.0, 1.0);

// Find color stops surrounding t and interpolate
for i in 0..(stop_count - 1) {
    if t >= stops[i].position && t <= stops[i+1].position {
        let local_t = (t - stops[i].position) / (stops[i+1].position - stops[i].position);
        return mix(stops[i].color, stops[i+1].color, local_t);
    }
}
```

### Expected Usage
```
Define 2+ color stops with positions [0, 1]
Set start and end points
Draw quad covering gradient area
```

### Output
Smoothly interpolated linear gradient.

## Integration with Intel EU Backend (Phase 4.3)

These shaders will be compiled to Intel EU machine code using the following pipeline:

```
WGSL source
    ↓ (naga::front::wgsl::parse_str)
naga IR Module
    ↓ (naga::valid::Validator)
Validated IR
    ↓ (Phase 4.3: Custom Intel EU backend)
Intel EU assembly
    ↓ (EU assembler)
GPU kernel binary
    ↓
3DSTATE_PS / 3DSTATE_VS commands
```

### Resource Binding Translation

WGSL bindings map to Intel surface state as follows:

| WGSL | Intel 3D State |
|------|----------------|
| `@group(0) @binding(0) var texture` | Surface state index 0 (RENDER_SURFACE_STATE) |
| `@group(0) @binding(1) var sampler` | Sampler state index 0 |
| `@group(1) @binding(0) var<uniform>` | Constant buffer at surface state index 1 |
| Push constants | Inline data in 3DPRIMITIVE command |

### Vertex Format
All shaders currently use procedural quad generation (no vertex buffer required). For positioned rendering, the vertex format will be:

```c
struct Vertex {
    float position[2];  // XY in normalized device coordinates
    float uv[2];        // Texture coordinates
};
```

Mapped to `3DSTATE_VERTEX_ELEMENTS`:
- Element 0: Position (R32G32_FLOAT, offset 0)
- Element 1: UV (R32G32_FLOAT, offset 8)

## Testing

All shaders are validated by the test suite in `render-sys/src/shader.rs`:

```bash
# Run all shader validation tests
cd render-sys && cargo test shader --target x86_64-unknown-linux-musl -- --nocapture

# Test specific shader
cd render-sys && cargo test test_sdf_text_shader --target x86_64-unknown-linux-musl -- --nocapture
```

Each test:
1. Loads the WGSL source
2. Parses it with naga
3. Validates the IR
4. Checks for required entry points (`vs_main`, `fs_main`)
5. Verifies shader stage compatibility
